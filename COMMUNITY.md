# Community

This repo is intentionally documentation-first: each skill is a self-contained folder under `skills/<name>/`.

## How To Contribute
- Fixes: clarify a checklist step, improve an output template, add a missing edge case.
- New skills: propose a niche skill with a concrete deliverable (templates/checklists) and a clear differentiator.

## Contribution Workflow
1. Open an issue describing the change.
2. Submit a PR with:
   - What changed and why
   - How you validated skill discovery: `npx skills add . --list --full-depth`

## What We Optimize For
- Independence: the skill can be used as-is without extra tooling.
- Practicality: the skill produces an artifact (spec, checklist, report template).
- Differentiation: avoid generic advice; focus on a specific workflow and deliverable.
