# Feedback

The fastest way to improve these skills is to collect small, specific feedback from real usage.

## What Feedback Is Most Useful
- A concrete task you tried to do with a skill
- Where the skill failed or was ambiguous
- What output template you expected but did not get
- What tools/stack you were using

## How To Share
If GitHub Issues are enabled, open an issue with:
- Skill name
- Scenario
- Expected output
- Actual output

## Quick Feedback Prompts
```text
Skill used:
Goal:
What worked:
What didn't:
Missing template/checklist:
1 change that would make it 2x better:
```
